Top 10 Herramientas para Junior Developer Web

Angel Campo Feb 02, 2019

En este artículo, vamos a ver las 10 mejores herramientas que todo desarrollador web junior debe tener en su cinturón. Con el conocimiento adecuado de cómo utilizar cada una de estas 10 herramientas, estarás bien encaminado para convertirte en un desarrollador web competente.

1. HTML, CSS y JavaScript

Ningún desarrollador web tendría esperanzas de tener éxito sin un buen dominio de HTML, CSS y JavaScript. Si aún no lo sabes, estos tres lenguajes forman la base de cada aplicación web que visitas en línea. HTML es lo que proporciona el contenido de los sitios web que visitas, como el texto que estás leyendo ahora. Por otro lado, CSS proporciona el estilo de estas páginas web, como el color de este texto. JavaScript es otra historia, y se puede utilizar para todo, desde la gestión del back-end de un sitio web con tecnologías como Node.js a la construcción de potentes visualizaciones de datos con D3.js. Saber cómo crear una aplicación web utilizando HTML, CSS y JavaScript será de gran beneficio para tu carrera como desarrollador web. Tener una base sólida en estos tres idiomas le permitirá comprender los procesos detrás de desarrollo web más íntimamente, y le permitirá crear sitios web que mucho más eficaz.

2. Bootstrap

En la última sección mencionamos CSS. Bootstrap es un framework CSS front-end mobile-first que mejorará en gran medida el tiempo que tardas en desarrollar una aplicación web. La mayoría de los estilos que necesitarás implementar como desarrollador web ya habrán sido implementados cientos de veces en el pasado. Bootstrap CSS toma muchos de estos estilos comunes y los empaqueta en un marco fácil de usar en el que sólo tienes que introducir el nombre de una clase para aplicar el estilo deseado. Un ejemplo de un estilo que han incluido en su marco es un sistema de cuadrícula que te permite establecer fácilmente diseños adaptables en tu sitio web. Cerca de 300 millones (alrededor del 20%) de los sitios web utilizan Bootstrap hoy en día, lo que lo convierte en una parte esencial de su kit de herramientas omo desarrollador web.

3. Node.js

Node.js es un framework JavaScript de back-end que está ganando popularidad rápidamente. Sitios web como LinkedIn, Netflix y Alibaba ya utilizan Node.js en su back-end. Aproximadamente 30 millones de sitios web de todo el mundo también utilizan Node.js. Aunque puede que no sea tan popular como Bootstrap CSS, está arrasando rápidamente en el mundo del desarrollo web. Lanzado en 2009, el proyecto Node.js alcanzó los mil millones de descargas en 2018. Si bien puede que no sea una herramienta esencial para tener en su haber al solicitar puestos de desarrollo web junior, los empleadores definitivamente lo considerarán un activo enorme si tiene conocimientos sobre el tema. Además de aprender Node.js, también te beneficiará aprender otro framework de JavaScript como React, Angular o Vue. Aunque cada uno de estos frameworks tiene diferentes puntos fuertes, en general todos se pueden utilizar para el mismo propósito. React y Vue son mejores para aplicaciones más ligeras, mientras que Angular es mejor para aplicaciones más complejas y preparadas para empresas. Vue se recomienda normalmente para principiantes, ya que algunos consideran que es más fácil de aprender que React.

4. MySQL o NoSQL

Independientemente de que ocupes o no un puesto de Front-end, saber cómo consultar una base de datos te resultará muy útil cuando trabajes en aplicaciones más grandes que tengan un componente de back-end. La mayoría de los empleadores también considerarán el conocimiento en esta área como un activo, ya que puede haber ocasiones en las que te preguntarán si serías capaz de resolver un problema con el back-end. Mientras que MySQL sigue siendo la tecnología de bases de datos más popular, con un 38,9% de todos los sitios web, MongoDB está cerrando rápidamente esta brecha para el lado NoSQL de las cosas y actualmente se encuentra en el 24,6%. Otro aspecto de MongoDB que algunos consideran un beneficio es el hecho de que utiliza el lenguaje de programación Python, que es ampliamente considerado como el lenguaje más fácil de aprender para los principiantes.

5. GIT

Ninguna lista de herramientas de desarrollo web estaría completa sin mencionar Git. No se puede exagerar la importancia del control de versiones cuando se trabaja en proyectos de cualquier tamaño sustancial. Sin tener un sistema de control de versiones en su lugar, muy rápidamente se encontrará perdiendo el tiempo cuando se trata de revertir los cambios. Saber cómo revertir a un commit anterior en lugar de reescribir el código te ahorrará horas a lo largo de tu carrera. Además, la mayoría de los empleadores consideran que Git es un prerrequisito para el trabajo. Aunque no se espera que tengas un conocimiento experto de Git cuando solicites un puesto junior, al menos ser capaz de entender el proceso básico de inicializar un proyecto, crear una rama, confirmar código y revertir a una confirmación anterior mejorará drásticamente tus posibilidades de conseguir un trabajo.

6. Jira

Jira es una herramienta que se utiliza en la mayoría de los lugares de trabajo que siguen la metodología de desarrollo ágil. Para aquellos de ustedes que no están familiarizados con el desarrollo ágil, esencialmente gira en torno a dividir un proyecto en pasos cada vez más pequeños organizados en sprints semanales. El principal punto focal de Jira es su tablero Kanban que muestra todas las diferentes tareas que necesitan ser hechas, en progreso, o ya terminadas. Comprender el flujo de trabajo del desarrollo ágil y, más concretamente, Jira, se considerará una ventaja para la mayoría de los empleadores.

7. Figma

Figma es una herramienta excelente para diseñar sitios web. Incluso si usted realmente no hace ningún diseño web en su papel como desarrollador web, la comprensión de cómo diseñar sitios web será de gran beneficio para usted no sólo en su carrera, sino también en sus proyectos personales también. Por cada minuto que dediques al diseño ahorrarás una cantidad considerable de tiempo en el desarrollo. Figma es mi herramienta de cabecera para todas mis necesidades de diseño de aplicaciones. Además, ¡es gratis!

8. Unsplash

Cuando diseñas páginas web, a menudo te encuentras buscando distintos tipos de imágenes para decorarlas. Cualquier desarrollador web que haya pasado por este proceso debe entender que no puede coger cualquier imagen de Internet y ponerla en su sitio web sin pagar derechos de autor o la atribución adecuada. Unsplash es un sitio web especializado en recopilar imágenes libres de derechos que puedes utilizar en tus proyectos. Yo lo utilizo no sólo para mis sitios web, sino también para mis artículos.

9. Flaticon

Al diseñar páginas web, además de las imágenes de mayor tamaño que necesitarás, en ocasiones también buscarás iconos que puedas utilizar. Ya sea para imágenes de perfil, iconos de redes sociales o cualquier otra cosa, se planteará el mismo problema de búsqueda de imágenes libres de derechos. Flaticon ofrece a los desarrolladores una amplia selección de iconos libres de derechos que pueden utilizar en sus proyectos.

10. Google

Aunque es posible que un empleador no te pregunte nunca: "¿Sabes utilizar Google?", saber no sólo cómo utilizarlo, sino cuándo hacerlo, te será de gran utilidad como desarrollador web. Si tienes un error, puedes ponerlo en Google y encontrar enlaces de problemas de Stack Overflow o GitHub con gente hablando de ese error exacto. Si estás buscando una herramienta para usar, de nuevo, Google será tu amigo. Independientemente de lo que busques cuando trabajes como desarrollador web, lo más probable es que Google sea tu punto de partida. Tener conocimientos sobre Google Dorks también puede ayudarte como desarrollador web. Mediante el uso de "dorks" puedes limitar los resultados de búsqueda a sitios web específicos, o incluso por tipo de archivo específico. Si sabes qué es lo que estás buscando específicamente, pero tus términos de búsqueda no están haciendo que el enlace se posicione, esto puede ayudarte a limitar los otros enlaces que están saturando los resultados.
